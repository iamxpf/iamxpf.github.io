web框架搭建——django的使用

django框架

采用MTV模型

M：模型，负责业务对象和数据库之间的映射

T模板，把页面展示给用户

V：视图，负责业务逻辑，可调用模型和模板

此外，还需要url分发器，将url请求分发给视图处理，视图再调用模型和模板

使用pycharm新建django项目如下：

其中manage.py为项目入口（其中包含了项目设置从何处读入，即需要去setting下进行查找），templates文件夹下为模板，项目名字下的urls.py为url分发器，需要自行新建应用app，使用app下的model.py，设置数据库中表的名称和各列的数据类型。

连接数据库——django与数据库连接

系统设置

（1）导入依赖

与mysql数据库相连，需要导入pymysql依赖：pip3 install pymysql

同时在项目同名下的__ init __.py下设置导入pymysql及版本

（2）在项目同名的文件夹下的setting设置连接的数据库的名字、端口、用户名、密码，即连接上了clTest数据库

如何与数据库连接

（1）利用ORM（对象关系映射）

django模型使用自带的ORM（对象关系映射），ORM将python对象和sql语句互相转换，使得项目能够访问sql数据库，ORM中的models类对应数据库中的表，对象实例对应表中的一条记录，属性对应表中的字段，对应关系如下：

（2）建立app

django中要使用模型就需要创建app

在终端输入python3 manage.py startapp bajie，同时在系统设置setting下加入app的名字，则创建了应用程序为bajie的app

（3）通过模型在数据库中创建表

利用app下的models.py创建类（对应就会在数据库中创建表）:设置user表，其中有对象有两个属性（对应两个字段），长度分别为20，10，都是字符类型

通过：python3 manage.py makemigrations 让django知道模型的变化

通过：python3 manage.py migrate 创建表结构（项目会根据model.py下设置的表结构在对应数据库下创建表）

打开数据库clTest，可以看到使用该应用程序创建的表（其余的都是系统表）

以上就是使用django框架搭建web应用，同时连接数据库，主要使用了django框架下的MTV模型中的M，即模型，下一个笔记主要记录如何使用V，即视图，使用数据库中的数据，并应用相关推荐算法得到推荐内容，并将结果通过T，即模板展示出来。
