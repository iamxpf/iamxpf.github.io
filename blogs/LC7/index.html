题目：402. 移掉K位数字

描述：给定一个以字符串表示的非负整数 num，移除这个数中的 k 位数字，使得剩下的数字最小。

注意:

num 的长度小于 10002 且 ≥ k。
num 不会包含任何前导零。
示例 1 :

输入: num = “1432219”, k = 3
输出: “1219”
解释: 移除掉三个数字 4, 3, 和 2 形成一个新的最小的数字 1219。

思路：从最高位开始，每次找到第一个比前面一个数字大的位置，移除该数字，此过程执行k次。

（1）将原始字符串复制给结果字符串中

（2）遍历字符串，找到第一个比前面一个数字大的位置，移除该位置的字符

（3）当移除完字符后，字符串首位为0，则将首位的字符删除，直至首位不是0为止

（4）以上两步（2）、（3）步骤执行k次，最终得到移除k个数字后的最小的数字

题目：1122. 数组的相对排序

描述：

给你两个数组，arr1 和 arr2，

arr2 中的元素各不相同
arr2 中的每个元素都出现在 arr1 中
对 arr1 中的元素进行排序，使 arr1 中项的相对顺序和 arr2 中的相对顺序相同。未在 arr2 中出现过的元素需要按照升序放在 arr1 的末尾。

示例：

输入：arr1 = [2,3,1,3,2,4,6,7,9,2,19], arr2 = [2,1,4,3,9,6]
输出：[2,2,2,1,4,3,3,9,6,7,19]

思路：类似桶排序的思想

（1）使用排序哈希表，存储数组arr1中每个数字-出现次数的映射

（2）遍历数组arr2，找到该数字在哈希表中出现的次数，按照出现的次数插入到结果数组中

（3）经过步骤（2）此时已经将 arr1 中项的相对顺序和 arr2 中的相对顺序相同，剩下处理未在 arr2 中出现过的元素

遍历哈希表，当当前元素出现次数不为0，则说明是没有在arr2中出现，按照出现的次数插入到结果数组中（因为哈希表排序过了，所以此时是按照升序放在 arr1 的末尾的。）

题目：1030. 距离顺序排列矩阵单元格

描述：给出 R 行 C 列的矩阵，其中的单元格的整数坐标为 (r, c)，满足 0 <= r < R 且 0 <= c < C。

另外，我们在该矩阵中给出了一个坐标为 (r0, c0) 的单元格。

返回矩阵中的所有单元格的坐标，并按到 (r0, c0) 的距离从最小到最大的顺序排，其中，两单元格(r1, c1) 和 (r2, c2) 之间的距离是曼哈顿距离，|r1 - r2| + |c1 - c2|。（你可以按任何满足此条件的顺序返回答案。）

示例 1：

输入：R = 1, C = 2, r0 = 0, c0 = 0
输出：[[0,0],[0,1]]
解释：从 (r0, c0) 到其他单元格的距离为：[0,1]

思路：

1、自定义排序 时间复杂度O(RClogRC)（排序时间RClogRC），空间复杂度O(logRC)（需要额外的栈存储）

（1）将横纵坐标与原点的差值以及当前位置保存下来

（2）对差值之和进行自定义排序

（3）按照排序之后的数组，保存第2和第3个索引上的位置

2、优先队列 时间复杂度O(RC)，空间复杂度O(RC)（需要额外的存储）

默认情况下，优先队列是大顶堆（降序），队头元素最大，要得到小顶堆（升序）需要添加保存数据的容器以及比较方式greater<>

（1）定义小顶堆，保存距离以及当前位置

（2）在数据插入小顶堆的过程中，就按照距离从小到大进行了排序

（3）只需要从队首一个个取出当前距离下的位置即可

3、桶排序 时间复杂度O(RC)，空间复杂度O(RC)（需要额外的桶存储）

（1）定义一个桶用于存储所有的距离

（2）遍历所有的坐标，得到当前坐标下的距离，按照距离的大小作为索引存储位置

则最终得到的桶是按照距离排序，每个距离桶里存储了当前得到该距离的位置

（3）遍历桶，得到每个桶下的位置进行保存即可

题目：面试题 10.01. 合并排序的数组

描述：给定两个排序后的数组 A 和 B，其中 A 的末端有足够的缓冲空间容纳 B。 编写一个方法，将 B 合并入 A 并排序。

初始化 A 和 B 的元素数量分别为 m 和 n。

示例:

输入:
A = [1,2,3,0,0,0], m = 3
B = [2,5,6], n = 3

输出: [1,2,2,3,5,6]

思路：

1、合并后用sort排序 时间复杂度O((m+n)log(m+n))，空间复杂度O(log(m+n))

2、双指针，分别指向两个数组的首部，比较对应元素，小的那个放到新数组中，移动指针，比较下一个，依次循环

时间复杂度O(m+n)，空间复杂度O(m+n)

3、三指针 从尾部开始比较 原地算法 时间复杂度O(m+n)，空间复杂度O(1)

（1）定义三个指针，分别指向数组A的尾部，数组A的最后一个元素，数组B的尾部

（2）当数组A和数组B的指针没有移动到首部时，继续循环

比较两个元素中大的那个，将其值赋值给数据A尾部的指针，并移动指针

（3）当循环结束后，数组B中还有元素，则将其全部赋值给数组A尾部的指针

（为什么不需要判断数组A中的元素：因为如果数组A中还有没有比较的元素，直接不处理，因为是原地算法）

题目：414. 第三大的数

描述：给定一个非空数组，返回此数组中第三大的数。如果不存在，则返回数组中最大的数。要求算法时间复杂度必须是O(n)。

示例 1:

输入: [2, 2, 3, 1]

输出: 1

解释: 注意，要求返回第三大的数，是指第三大且唯一出现的数。
存在两个值为2的数，它们都排第二。

思路：

1、每次输出当前最大的值

（1）当当前数组内元素没有遍历结束，继续循环

（2）找到数组当前最大的值，删除该值，并进行最大值赋值

（3）如果当前最大值和上一次一样，不加入计数中，反之进行计数，并重新设置最大值

（4）当计数值等于3，则弹出，返回当前第三大的值

（5）当遍历结束，计数值小于3，则重新遍历，找到原数组中的最大值返回

2、利用set，维护大小为3的set（默认排序是从小到大）

（1）遍历数组中的元素将其插入set中（此时自动去重）

当set的大小大于3时，删除最前面一个元素

（2）当最后得到的set大小等于3，直接返回最前面一个元素的值，即第三大的值（s.begin）

当大小小于3，返回数组内最大的值，即最后一个元素的值，即s.rbegin()

注：

c.begin() 返回一个迭代器，它指向容器c的第一个元素

c.end() 返回一个迭代器，它指向容器c的最后一个元素的下一个位置

c.rbegin() 返回一个逆序迭代器，它指向容器c的最后一个元素

c.rend() 返回一个逆序迭代器，它指向容器c的第一个元素前面的位置

题目：509. 斐波那契数

描述：斐波那契数，通常用 F(n) 表示，形成的序列称为斐波那契数列。该数列由 0 和 1 开始，后面的每一项数字都是前面两项数字的和。也就是：

F(0) = 0, F(1) = 1
F(N) = F(N - 1) + F(N - 2), 其中 N > 1.
给定 N，计算 F(N)。

示例 1：

输入：2
输出：1
解释：F(2) = F(1) + F(0) = 1 + 0 = 1.

思路：动态规划

设置dp数组大小为N+1，保存从0开始的N+1个数字的斐波那契和

（1）定义初始状态

F(0) = 0, F(1) = 1

（2）定义转移方程

F(N) = F(N - 1) + F(N - 2)

最后返回dp数组最后一个值即为F(N)的结果

题目：283. 移动零

描述：给定一个数组 nums，编写一个函数将所有 0 移动到数组的末尾，同时保持非零元素的相对顺序。

示例:

输入: [0,1,0,3,12]
输出: [1,3,12,0,0]
说明:

必须在原数组上操作，不能拷贝额外的数组。
尽量减少操作次数。

思路：

1、两次遍历：非0元素前移，0元素放置最后

（1）第一次遍历数组：找到非0的值前移即可

（2）第二次遍历数组：在非0元素最后加上0

2、一次遍历：双指针，保证左指针的左边为非0，右指针和左指针之间为0

（1）设置两个指针都指向首部

（2）当移动右指针不为0时，将左右指针的值交换，保证左指针左边非0，移动左右指针

当右指针为0，则继续找不为0的值

题目：496. 下一个更大元素 I

描述：给定两个 没有重复元素 的数组 nums1 和 nums2 ，其中nums1 是 nums2 的子集。找到 nums1 中每个元素在 nums2 中的下一个比其大的值。

nums1 中数字 x 的下一个更大元素是指 x 在 nums2 中对应位置的右边的第一个比 x 大的元素。如果不存在，对应位置输出 -1 。

示例 1:

输入: nums1 = [4,1,2], nums2 = [1,3,4,2].
输出: [-1,3,-1]
解释:
对于num1中的数字4，你无法在第二个数组中找到下一个更大的数字，因此输出 -1。
对于num1中的数字1，第二个数组中数字1右边的下一个较大数字是 3。
对于num1中的数字2，第二个数组中没有下一个更大的数字，因此输出 -1。
示例 2:

输入: nums1 = [2,4], nums2 = [1,2,3,4].
输出: [3,-1]
解释:
对于 num1 中的数字 2 ，第二个数组中的下一个较大数字是 3 。
对于 num1 中的数字 4 ，第二个数组中没有下一个更大的数字，因此输出 -1 。

思路：

1、暴力法：遍历nums1中的每个元素，首先找到其在nums2中所在的位置，再判断该位置之后有无比其更大的元素 时间复杂度O(m * n )(m为nums1大小，n为nums2大小)，空间复杂度O(1)

（1）遍历nums1中的每个元素

（2）找到其在nums2中所在的位置

（3）判断该位置之后有无比其更大的元素

有则保存比其大的数，没有就保存-1

2、单调栈：对nums2维护一个单调栈，保存每个元素和其下一个更大的元素的映射 时间复杂度O(m+n )(m为nums1大小，n为nums2大小)，空间复杂度O(n)（用到的哈希表和栈所占的空间）

（1）对nums2维护一个单调栈，保存每个元素和其下一个更大的元素的映射 ：单调栈内的元素是从栈顶到栈底是升序排列，即栈底的元素大于上面的元素

先将第0个元素压入单调栈中，遍历nums2中的每个元素（从第1个元素开始），当当前元素比栈顶的元素大，则栈内所有元素的下一个更大的值为该遍历到的元素（因为入栈是则说明之前都没有遇到更大的元素），将其保存在哈希表中；

反之，即当前元素比栈顶元素小，则入栈，等待后面的元素比其大

（2）当遍历结束，单调栈内还有元素：说明栈内的元素到最后都没有遇到比它大的元素：则将栈内所有元素都映射到-1

（3）遍历nums2数组，找到每个元素在哈希表中的映射，即为下一个更大的元素

题目：541. 反转字符串 II

描述：给定一个字符串 s 和一个整数 k，你需要对从字符串开头算起的每隔 2k 个字符的前 k 个字符进行反转。

如果剩余字符少于 k 个，则将剩余字符全部反转。
如果剩余字符小于 2k 但大于或等于 k 个，则反转前 k 个字符，其余字符保持原样。

示例:

输入: s = “abcdefg”, k = 2
输出: “bacdfeg”

思路：

1、分组法 时间复杂度O(n)，空间复杂度O(n)（用到额外字符串数组所占的空间）

（1）将原数组按照每2k个进行分组，存入到字符串数组中

设置起始位置，当当前位置向后2k个字符没有超出范围，则将从起始位置开始的2k个字符存储下来；

反之如果超出范围，则将当前位置到末尾的字符串存储下来

（2）读入分组存储的字符串数组

如果长度大于等于k，则反转前k个字符；

反之，即长度小于k，则将整个字符串反转

处理完成后保存到结果中即可。

2、优化1

直接在1中每次分组的时候反转前k个字符即可

（1）确定起始位置，控制步长为2 * k

（2）当从起始位置开始的k个位置不超出范围，则进行反转；

反之，则将剩余的字符串全部反转

题目：147. 对链表进行插入排序

描述：插入排序的为：从第一个元素开始，该链表可以被认为已经部分排序（用黑色表示）。
每次迭代时，从输入数据中移除一个元素（用红色表示），并原地将其插入到已排好序的链表中。

插入排序算法：

插入排序是迭代的，每次只移动一个元素，直到所有元素可以形成一个有序的输出列表。
每次迭代中，插入排序只从输入数据中移除一个待排序的元素，找到它在序列中适当的位置，并将其插入。
重复直到所有输入数据插入完为止。

示例 1：

输入: 4->2->1->3
输出: 1->2->3->4

思路：遍历链表，找到破坏升序的位置，再从头找到让其插入的位置 时间复杂度O(n * n)，空间复杂度O(1)（原地算法）

（1）设置哑结点，next指针指向原链表

（2）遍历链表

当当前节点的值比下一个节点的值小，则继续比较下一个节点；

当当前节点的值比下一个节点的值大，则找到了需要找到其插入位置的节点cur=当前节点的下一个节点：

利用哑结点从头开始遍历，找到其需要插入的位置，将当前节点指向下下个节点，cur节点指向需要插入的位置，并将哑结点当前位置的下一个设置为cur

（3）返回哑结点的下一个

题目：LCP 01. 猜数字

描述：小A 和 小B 在玩猜数字。小B 每次从 1, 2, 3 中随机选择一个，小A 每次也从 1, 2, 3 中选择一个猜。他们一共进行三次这个游戏，请返回 小A 猜对了几次？

输入的guess数组为 小A 每次的猜测，answer数组为 小B 每次的选择。guess和answer的长度都等于3。

示例 1：

输入：guess = [1,2,3], answer = [1,2,3]
输出：3
解释：小A 每次都猜对了。

示例 2：

输入：guess = [2,2,3], answer = [3,2,1]
输出：1
解释：小A 只猜对了第二次。

思路：遍历数组，对应位置相同则计数加1，最后返回计数值即可

题目：1539. 第 k 个缺失的正整数

描述：给你一个 严格升序排列 的正整数数组 arr 和一个整数 k 。

请你找到这个数组里第 k 个缺失的正整数。

示例 1：

输入：arr = [2,3,4,7,11], k = 5
输出：9
解释：缺失的正整数包括 [1,5,6,8,9,10,12,13,…] 。第 5 个缺失的正整数为 9 。

思路：

（1）设置i从1开始，并且遍历数组内所有元素

当数组内元素和i不相等时，继续计数，并移动i，同时判断计数值是否等于k，相等则返回当前的i值即缺失的值

当组内元素和i相等，则移动i，遍历下一个元素

（2）当遍历到数组末尾，计数值仍然小于k，则说明缺失值在末尾的值之后

将数组最后一个值++，直至计数值等于k

题目：384. 打乱数组

描述：给你一个整数数组 nums ，设计算法来打乱一个没有重复元素的数组。

实现 Solution class:

Solution(int[] nums) 使用整数数组 nums 初始化对象
int[] reset() 重设数组到它的初始状态并返回
int[] shuffle() 返回数组随机打乱后的结果

示例：

输入
[“Solution”, “shuffle”, “reset”, “shuffle”]
[[[1, 2, 3]], [], [], []]
输出
[null, [3, 1, 2], [1, 2, 3], [1, 3, 2]]

思路：

（1）初始化

可以直接设置一个私有变量，将传入的数组的值赋值给它

（2）重置

将原来的私有变量的值返回

（3）打乱

1.暴力法 时间复杂度O(n^2) (乘方时间来自erase)，空间复杂度O(n)（用到额外字符串数组所占的空间）

（1）每次从数组中随机选择一个位置赋值给新数组

（2）为了避免重复，每次选择的数字需要删除

一直到新数组被填满

2.洗牌算法 时间复杂度O(n)，空间复杂度O(n)（用到额外数组所占的空间）

每次从前i-1张洗好的牌中，选择一张和当前第i张牌交换即可
